Why it exists

Apple Notes is convenient. Local Markdown is durable.

Apple Notes is excellent for quick capture, phone-first editing, and notes shared with family or teammates. But when the long-term version of your notes needs to be searchable, scriptable, versioned, and ready for AI workflows, local Markdown still wins.

NotesBridge is built for people who want both: keep writing in Apple Notes, then turn that note graph into a local-first workspace without throwing away folders, attachments, front matter, or internal links.

Project snapshot

The repo is public. The app is practical. The design stays out of the way.

What it does today

Two jobs, one bridge.

Editing layer

Enhance the macOS editing experience without replacing Apple Notes.

  • Slash commands with inline exact-match execution
  • Markdown-style triggers for lists and structure
  • Floating formatting toolbar after text selection
Sync output

Export to local folders that stay useful outside the app.

  • Folder structure preserved
  • Native attachments exported
  • Front matter and internal links retained
AI-ready workspace

Make shared notes easier to organize, automate, version, and inspect.

  • Works with local-first vault-style workflows
  • Better handoff into Git, search, and agent tooling
  • Useful even when Apple Notes remains the capture layer

Workflow

Use Apple Notes for input, keep Markdown for the durable copy.

  1. Install and grant permissions. NotesBridge runs as a lightweight menu bar app and asks for the macOS permissions needed for inline enhancements and full-note sync.
  2. Point the app at the Apple Notes data folder on first full sync. That lets NotesBridge read the Notes database and attachment files directly from `group.com.apple.notes`.
  3. Keep writing in Apple Notes while the exported workspace stays local. The result is a Markdown folder structure that is easier to inspect, automate, and collaborate on with AI agents.

Current constraints

Deliberately conservative where Apple Notes stays closed.

  • Apple Notes exposes no public plugin API, so NotesBridge is a companion app, not an in-app extension.
  • Inline enhancements depend on Accessibility and event synthesis.
  • The current sync direction is Apple Notes -> local folders, with no reverse sync.
  • Slash commands currently favor exact command plus space and mouse-driven suggestion selection.

Q&A

The questions people usually ask first.

Is NotesBridge a plugin inside Apple Notes?

No. Apple Notes does not expose a public plugin or extension API, so NotesBridge runs as a separate macOS companion app.

Does it sync back into Apple Notes?

Not today. The current design is intentionally one-way: Apple Notes -> local Markdown folders.

Why not just export from Apple Notes manually?

The native export path is weak for ongoing workflows. NotesBridge is built around repeated sync, folder preservation, attachment export, and keeping the output useful for local-first tooling.

Does it work with shared notes?

Yes. Shared Apple Notes are part of the intended workflow. The goal is to turn that shared input into a Markdown workspace that stays local and easier to process afterward.

What survives the sync?

The current export keeps folder structure, native attachments, front matter metadata, and internal links. That is the core product shape.
